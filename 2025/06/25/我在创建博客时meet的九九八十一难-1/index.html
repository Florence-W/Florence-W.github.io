我在创建博客时meet的九九八十一难

发表于2025-06-25|更新于2025-06-25
|浏览量:

1.问题：git克隆下载butterfly主题失败（原因：网络连接错误）
解决方案：手动下载ZIP安装：将下载的主题安装包名字改成butterfly然后移动到博客目录下themes文件夹中，将D:\wblog\blog_config.yml 文件中的这一行修改成这样theme: butterfly。
2.问题：第一次启动页面乱码（原因：可能是文件编码格式不对 或者配置或网页使用了不支持中文的字体）
解决方案：我首先确认了Hexo配置文件编码为UTF-8（这个问题我之前做跑前端的代码也出现过 就是这样的弄好的）但是再次打开还是乱码，然后我修改了butterfly主题配置文字：修改butterfly下的_config.yml文件的字体部分为支持中文的字体，保存 再次运行后成功。
3.问题：在VScode中无法打开.md文件 但是在_posts文件夹中是有的
解决方案：直接在文件夹中打开.md文件
4.问题：博客仓库成功创建 但是部署不成功
解决方案：首先检查_config.yml 中的 deploy 设置是否正确？（其中repo部分需特别注意）然后检查是否已经安装了 Hexo 的部署插件？然后在终端输入指令清除缓存，生成静态页面，部署到Gitub，正常部署成功的话，我的仓库中会出现html文件但是我的仓库并没有出现，我开始尝试在 Windows 上配置 SSH 密钥并将其添加到 GitHub。
5.问题：我的SSH密钥生成失败，是因为路径中包含了 中文用户名「吴海鑫」
解决方案：我先是手动创建.ssh文件夹 又失败了。。。，又指定英文路径保存密钥文件成功生成密钥。
6.问题：Git Bash 无法在我的用户文件夹中创建 .ssh 目录来保存 known_hosts，导致 SSH 验证失败，回退成密码方式（但我没设置密码，所以一直失败）–实际上还是我的用户文件路径包含中文
解决方案：手动创建.ssh目录然后把密钥转移到.ssh文件夹，添加私钥到SSH agent后再次连接最后SSH 密钥也成功添加并注册到 Git Bash中
7.心得：
几经波折……（实际上是一步一个坎，各种小小怪怪的问题）
好在最后成功了！
